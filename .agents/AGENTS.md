# Project-Scoped Rules
## MCP Tool Limitations
- **Expose via User**: When encountering limitations running MCP tools or executing commands (such as lacking explicit tool declarations for background operations or launch tasks), do NOT modify the MCP server repository or write local temporary scripts within it to bridge the functionality. Instead, describe the limitation and ask the user to expose the necessary API endpoints or MCP definitions.
## Repo-MCP Access Restriction
- **Rule**: When working in any RimSynapse workspace, NEVER review, view, or read files under the `Repo-MCP` directory (`d:/github/rimsynapse/Repo-MCP`). Avoid inspecting its source code, scripts, or configuration.
